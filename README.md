# ПРОБЛЕМЫ РЕАЛИЗОВАННОГО КОНТЕНТА
Интересная идея — детерминированный генератор паролей (пароль = функция от «сервис + мастер-фраза»), в духе SuperGenPass. Но в реализации есть серьёзные проблемы, в основном связанные с безопасностью.
Критично: криптографическая часть
1. System.Random — не подходит для генерации паролей. Это некриптографический ГПСЧ с полностью предсказуемым алгоритмом. Если атакующий знает алгоритм (а он у вас open-source) и подберёт seed, он мгновенно восстановит весь пароль. Нужно генерировать байты через System.Security.Cryptography.RandomNumberGenerator, а не new Random(seed).
2. Огромная потеря энтропии при вычислении seed.
seed = Convert.ToInt32(GetHash(seedString).Substring(0, 10), 16);
MD5 даёт 128 бит, но вы берёте только первые 10 hex-символов (40 бит).
Convert.ToInt32 — это ещё и 32-битный int, так что реальный seed сжимается до 2^32 возможных значений.
Итог: даже при сложной мастер-фразе, пространство возможных паролей для конкретного сервиса — максимум 4 миллиарда вариантов. Это перебирается на обычном ПК за разумное время, если известны сервис и набор символов.
3. MD5 — устаревший хэш, но в данном контексте это не главная проблема (не коллизии страшны, а отсутствие key-stretching). Тут просится HMAC-SHA256 или, лучше, PBKDF2/Argon2 — с большим числом итераций, чтобы подбор мастер-пароля был дорогим.
4. Фоллбэк на текущее время, если seed пустой:
seed = (int)(DateTime.Now.Subtract(new DateTime(1970, 1, 1))).TotalSeconds;
Это уничтожает саму идею детерминированного генератора (тот же пароль в другой раз не сгенерировать) и вдобавок seed предсказуем (текущее время известно/угадывается с точностью до минут).
Что я бы сделал вместо этого:
using var hmac = new HMACSHA256(Encoding.UTF8.GetBytes(masterSecret));
byte[] hash = hmac.ComputeHash(Encoding.UTF8.GetBytes(serviceName));
// затем через rejection sampling мапить байты hash на symbols,
// при нехватке байт — дохэшировать (HKDF-подобно)
Так пароль зависит от полных 256 бит, без обрезки, и без некриптографического RNG.
Баги и мелкие проблемы
MD5.Create() не освобождается — оберните в using.
Конкатенация строк в циклах (Password = Password + ..., result += ...) — лучше StringBuilder, особенно при увеличении длины пароля.
Decimal.Parse(settings[0]) и обращения к settings[1..4] упадут с исключением, если settings.txt повреждён или в другом формате — нет try/catch.
История сервисов хранится в открытом виде в services.txt — это утечка информации о том, для каких сервисов у вас вообще есть пароли (не сам пароль, но метаданные).
Именование локальных переменных с большой буквы (Length, Password, SymbolsUsed) — не C#-конвенция, обычно camelCase для локальных переменных.
Что хорошо
Идея исключить визуально похожие символы (I, O, l, 0, 1 частично убраны) — разумно для юзабилити.
Разделение UI-логики и генерации достаточно чистое для WinForms-приложения.
Сохранение/восстановление настроек между сессиями удобно.
Главный вывод: сама концепция (детерминированный генератор от мастер-фразы) рабочая и используется в реальных менеджерах паролей, но текущая реализация серьёзно ослабляет её из-за усечения хэша до 32 бит и использования System.Random. Если это для реального использования (не учебный проект), я бы переписал ядро генерации на HMAC/PBKDF2 + CSPRNG.

# ПЛАН КОНВЕРТАЦИИ: C# → PYTHON (GUI → WinForms)

## Цель
Переписать существующее C# WinForms-приложение на Python, сохранив WinForms как технологию GUI.
В качестве прослойки используется **Python.NET (pythonnet)** — библиотека, позволяющая вызывать .NET-сборки
из Python напрямую (включая `System.Windows.Forms`).

## Этап 1: Окружение и зависимости

1. **Установка .NET SDK / Mono** — нужен рантайм, в котором живут WinForms-сборки.
   - На **Windows**: достаточно .NET SDK (6.0+) или .NET Framework (встроен).
   - На **Linux/Mac**: устанавливается `mono-complete` (реализация WinForms через Mono).
2. **Установка pythonnet**:
   ```bash
   pip install pythonnet
   ```
   — даёт модуль `clr`, который загружает .NET-сборки и делает их классы доступными из Python.
3. **Проверка**: написать минимальный скрипт, открывающий пустое WinForms-окно из Python.

## Этап 2: Портирование ядра генерации (C# → Python)

Перенести алгоритм генерации пароля из `MainForm.cs` в отдельный Python-модуль `core.py`:

1. **Хэш-функция** — `MD5(сервис + сид)` → Python `hashlib.md5()`.
2. **Вычисление seed** — `int(hash[:10], 16)`.
3. **ГПСЧ** — `random.Random(seed)` с тем же набором символов.
4. **Фоллбэк** на текущее время при пустом seed.
5. **Функция-обёртка**: `generate_password(service: str, seed: str, length: int, symbol_sets: dict) -> str`.
6. **Наборы символов** — те же: capital, lower, digits, symbols (с теми же исключениями).

Все изменения алгоритма (HMAC, key-stretching) отложены на отдельный этап рефакторинга.

## Этап 3: Портирование WinForms-интерфейса

Скопировать layout из `MainForm.Designer.cs` в Python через `clr`:

1. **Подключение сборок**:
   ```python
   import clr
   clr.AddReference('System.Windows.Forms')
   clr.AddReference('System.Drawing')
   from System.Windows.Forms import *
   from System.Drawing import *
   ```
2. **Создать класс `MainForm(Form)`** — наследник `System.Windows.Forms.Form`.
3. **Перенести все контролы**:
   - `tbSeed` → TextBox
   - `cmbService` → ComboBox (с историей)
   - `numLength` → NumericUpDown
   - `cbCapital`, `cbLower`, `cbDigits`, `cbSymbols` → CheckBox
   - `btnGenerate` → Button ("Готово")
   - `tbPassword` → TextBox (readonly, для вывода)
   - `btnCopy` → Button ("Скопировать")
4. **Привязать события**:
   - `Click` на кнопках
   - `KeyDown` на ComboBox (Enter → генерация)
   - `KeyDown` на tbPassword (Enter → копирование)
   - `Load` / `FormClosing` для загрузки/сохранения настроек

## Этап 4: Портирование сохранения данных

1. `services.txt` — список сервисов (ComboBox.Items) — загружается при старте, сохраняется при закрытии.
2. `settings.txt` — настройки: длина + флаги символов — формат CSV как в C#.
3. Обработка ошибок: try/except при парсинге settings.txt (защита от повреждённого файла).

## Этап 6: Рефакторинг безопасности

После портирования всей функциональности — переписать ядро:

1. MD5 → **HMAC-SHA256** (ключ = сид, сообщение = сервис).
2. Отказ от `Random(seed)` → генерация через **random.SystemRandom** или **os.urandom** + rejection sampling на основе HMAC-хэша.
3. Убрать 32-битное усечение seed — использовать полные 256 бит HMAC.
4. Опционально: **PBKDF2** с 100 000+ итераций для stretching мастер-фразы.
5. Убрать фоллбэк на `DateTime.Now` — вместо этого выбрасывать исключение или запрашивать ввод.
